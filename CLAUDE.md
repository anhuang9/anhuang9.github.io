## Project Overview

Personal portfolio website for a UCLA Computer Science student. The site is a static HTML/CSS/JS project hosted on GitHub Pages via Jekyll. It includes:

- A home/landing page with parallax scrolling
- A blog page (placeholder content, not yet fully implemented)
- A photography gallery with an auto-rotating carousel
- A projects showcase page
- A downloadable resume (PDF)

## Development Workflow

### Local Development

Serve the site locally with Jekyll:

```bash
# Install Jekyll (requires Ruby)
gem install jekyll bundler

# Serve locally
jekyll serve
# Site available at http://localhost:4000
```

Alternatively, since all pages are static, any local HTTP server works:

```bash
python3 -m http.server 8000
```

### Making Changes

Each page is self-contained:

- Edit the `.html` file for structure/content
- Edit the corresponding `.css` file for styles
- `photography.js` handles gallery carousel behavior only

Changes to `index.html` use `main.css`. Each other page has its own CSS file with matching names (`blog.css`, `photography.css`, `projects.css`).

### Deployment

Pushing to `master` automatically deploys to GitHub Pages. There is no manual build step.

### CI/CD

`.github/workflows/semgrep.yml` runs Semgrep static analysis on every push and pull request to `master`. It requires a `SEMGREP_APP_TOKEN` repository secret to publish results to the Semgrep App dashboard.

## Code Conventions

### HTML

- Bootstrap 4 grid and utilities are used for layout and responsiveness
- All pages share the same navbar structure — keep it consistent when editing
- The navbar uses `navbar-dark` with a custom cyan background (`#58deff`)
- Use `lang="en"` on `<html>` tags
- External libraries are loaded from CDN; do not introduce npm-managed dependencies

### CSS

- **Parallax effects** on `index.html` use CSS 3D transforms (`perspective`, `translateZ`) — do not simplify these without understanding the layout impact
- Typography uses `vw` units for viewport-relative scaling (e.g., `4.5vw` for headings)
- Color palette:
  - Cyan: `#58deff` (navbar, accents)
  - Red: `rgb(255, 0, 0)` (hover states)
  - White: default text on dark/image backgrounds
  - Semi-transparent backgrounds: `rgba(...)` for readability overlays
- CSS transitions are `0.25s`–`0.33s` ease for smoothness
- Font: Nunito (Google Fonts import at top of each CSS file)

### JavaScript

- ES6 syntax: use `const`/`let`, arrow functions
- The gallery in `photography.js` uses an Object.create() prototype pattern (Behavior Delegation)
- The carousel advances every 3 seconds using `setInterval`
- Slider position is controlled via a CSS custom property `--slider-offset`
- Keep JS minimal; prefer CSS for visual effects

### Assets

- Photography images live in `/background/` — add both original and `rsz_` (resized) versions when adding new photos
- Favicons are in `/favicomatic/` — regenerate via Favicomatic if changing the favicon
- PDFs (resume, etc.) live in the repository root

## Navigation Structure

All pages link to each other via a shared Bootstrap navbar:

```
Home (index.html) → Blog | Photography | YouTube | Projects | Resume | Contact
```

When adding a new page, update the navbar in **all** existing HTML files to stay consistent.

## Security

- Semgrep scans run automatically on PRs/pushes to `master`
- No user input, forms, or backend — attack surface is minimal
- CDN dependencies (Bootstrap, jQuery) should be checked for known vulnerabilities if upgrading
- Avoid inline event handlers (`onclick=...`) — use `addEventListener` in JS files
